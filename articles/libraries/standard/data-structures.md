### <a name="amplitude-amplification-oracles"></a><span data-ttu-id="3eac7-128">Amplitúdó-erősítés Oracles</span><span class="sxs-lookup"><span data-stu-id="3eac7-128">Amplitude Amplification Oracles</span></span> ###

<span data-ttu-id="3eac7-129">Az amplitúdó-erősítési algoritmus célja, hogy elvégezze a kezdeti állapot és a végső állapot közötti rotációs műveletet az állapot reflexiós sorrendjének alkalmazásával.</span><span class="sxs-lookup"><span data-stu-id="3eac7-129">The amplitude amplification algorithm aims to perform a rotation between an initial state and a final state by applying a sequence of reflections of the state.</span></span>

<span data-ttu-id="3eac7-131">Ezeket a specifikációkat két Oracle adja meg.</span><span class="sxs-lookup"><span data-stu-id="3eac7-131">These specifications are given by two oracles.</span></span>
<span data-ttu-id="3eac7-132">Ezek a jóslatok úgy működnek, hogy a bemeneteket két szóközre, egy "cél" alterületre és egy "kezdeti" alterületre bontja.</span><span class="sxs-lookup"><span data-stu-id="3eac7-132">These oracles work by breaking the inputs into two spaces, a "target" subspace and an "initial" subspace.</span></span>

<span data-ttu-id="3eac7-135">Azt is vegye figyelembe, hogy ez a két alterület általában nem zárják ki egymást: mindkét szóköz tagjai lesznek.</span><span class="sxs-lookup"><span data-stu-id="3eac7-135">Also note that these two subspaces are not usually mutually exclusive: there will be vectors that are members of both spaces.</span></span>
<span data-ttu-id="3eac7-136">Ha ez nem igaz, az amplitúdó-erősítésnek nincs hatása, ezért a kezdeti alterületnek nem kell átfedésben lennie a cél alterülettel.</span><span class="sxs-lookup"><span data-stu-id="3eac7-136">If this were not true then amplitude amplification would have no effect so we need the initial subspace to have non-zero overlap with the target subspace.</span></span>

<span data-ttu-id="3eac7-137">Azt az első Oracle-t jelöljük, amelyre az amplitúdó-erősítéshez szükség van, hogy $P\_$0, amely a következő művelettel rendelkezik.</span><span class="sxs-lookup"><span data-stu-id="3eac7-137">We will denote the first oracle that we need for amplitude amplification to be $P\_0$, defined to have the following action.</span></span>  <span data-ttu-id="3eac7-138">Minden, a "kezdeti" alterületen a $ \ket{x} $ értéknél $P\_0 \ket{x} =-\ket{x} $ és minden olyan állam számára, amely nem ebben az altérben van, $P\_0 \ket{y} = \ket{y} $.</span><span class="sxs-lookup"><span data-stu-id="3eac7-138">For all states $\ket{x}$ in the "initial" subspace $P\_0 \ket{x} = -\ket{x}$ and for all states $\ket{y}$ that are not in this subspace we have $P\_0 \ket{y} = \ket{y}$.</span></span>

<span data-ttu-id="3eac7-141">Hasonlóképpen, az összes olyan állam esetében, amely nem a \ket{y} $ $P _1 \ ket {y} = \ket{y} $.</span><span class="sxs-lookup"><span data-stu-id="3eac7-141">Similarly, for all states $\ket{y}$ that are not in the target subspace $P_1\ket{y} = \ket{y}$.</span></span>
<span data-ttu-id="3eac7-142">Ezt a két reflexiót egyesítjük úgy, hogy olyan operátort alkotnak, amely az amplitúdó erősítésének egyetlen lépését adja meg, $Q =-P_0 P_1 $, ahol a teljes mínusz jel csak az ellenőrzött alkalmazások esetében fontos.</span><span class="sxs-lookup"><span data-stu-id="3eac7-142">These two reflections are then combined to form an operator that enacts a single step of amplitude amplification, $Q = -P_0 P_1$, where the overall minus sign is only important to consider in controlled applications.</span></span>
<span data-ttu-id="3eac7-143">Az amplitúdó-erősítés ezután egy kezdeti állapotba kerül, amely a kezdeti alterületen található $ \ket{\psi} $, majd végrehajtja a $ \ket{\psi} \mapsto Q ^ m \ket{\psi} $ értéket.</span><span class="sxs-lookup"><span data-stu-id="3eac7-143">Amplitude amplification then proceeds by taking an initial state, $\ket{\psi}$ that is in the initial subspace and then performs $\ket{\psi} \mapsto Q^m \ket{\psi}$.</span></span>

<span data-ttu-id="3eac7-147">Ezért fontos, hogy az amplitúdó-erősítés a kvantum-számítástechnika számos alkalmazásának jelentős építőeleme.</span><span class="sxs-lookup"><span data-stu-id="3eac7-147">This is why amplitude amplification is a significant building block for many applications of quantum computing.</span></span>

<span data-ttu-id="3eac7-148">Az algoritmus használatának megismeréséhez hasznos lehet olyan példát biztosítani, amely az Oracle-ket is felépíti.</span><span class="sxs-lookup"><span data-stu-id="3eac7-148">In order to understand how to use the algorithm, it is useful to provide an example that gives a construction of the oracles.</span></span>  <span data-ttu-id="3eac7-149">Ebben a beállításban érdemes megfontolni a a kiinduló adatbázis-keresésekhez szükséges a a a a a</span><span class="sxs-lookup"><span data-stu-id="3eac7-149">Consider performing Grover's algorithm for database searches in this setting.</span></span>
<span data-ttu-id="3eac7-150">A a a \ket{+} ^ {\otimes n} = H ^ {\otimes n} \ket{0}$ értéket az egyik (potenciálisan) számos megjelölt állapotba alakítja a a következőben:.</span><span class="sxs-lookup"><span data-stu-id="3eac7-150">In Grover's search the goal is to transform the state $\ket{+}^{\otimes n} = H^{\otimes n} \ket{0}$ into one of (potentially) many marked states.</span></span>
<span data-ttu-id="3eac7-151">A további egyszerűsítés érdekében nézzük meg azt az esetet, amikor az egyetlen megjelölt állapot $ \ket{0}$.</span><span class="sxs-lookup"><span data-stu-id="3eac7-151">To further simplify, let's just look at the case where the only marked state is $\ket{0}$.</span></span>
<span data-ttu-id="3eac7-152">Ezt követően két Oracle-t tervezünk: az egyiket, amely csak az $ \ket{+} ^ {\otimes n} $ kezdeti állapotot jelöli mínusz jellel, egy másikat, amely a megjelölt állapotot ($ \ket{0}$) mínusz jellel jelöli.</span><span class="sxs-lookup"><span data-stu-id="3eac7-152">Then we have design two oracles: one that only marks the initial state $\ket{+}^{\otimes n}$ with a minus sign and another that marks the marked state $\ket{0}$ with a minus sign.</span></span>
<span data-ttu-id="3eac7-153">Az utóbbi kaput a következő eljárással lehet megvalósítani a Canon vezérlési folyamat műveleteinek használatával:</span><span class="sxs-lookup"><span data-stu-id="3eac7-153">The latter gate can be implemented using the following process operation, by using the control flow operations in the canon:</span></span>

```qsharp
operation ReflectAboutAllZeros(register : Qubit[]) : Unit 
is Adj + Ctl {

    // Apply $X$ gates to every qubit.
    ApplyToEach(X, register);

    // Apply an $n-1$ controlled $Z$-gate to the $n^{\text{th}}$ qubit.
    // This gate will lead to a sign flip if and only if every qubit is
    // $1$, which happens only if each of the qubits were $0$ before step 1.

    // Apply $X$ gates to every qubit.
    ApplyToEach(X, register);
}
```
